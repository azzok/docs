Command Library

Setup notes and copy-ready commands for the Azzok stack. Pick a tool, copy the command, and go.

Install dependencies

Run in the project root after cloning to pull Composer packages.

# from the Laravel project root
composer install

Environment & app key

Copy the sample env file, then generate the encryption key.

cp .env.example .env
php artisan key:generate

Run the dev server

Serves the app locally. Use 0.0.0.0 to reach it from a device on your network.

php artisan serve --host=0.0.0.0 --port=8000

Migrations

migrate:fresh drops all tables and re-runs — never use it in production.

php artisan migrate
php artisan migrate:fresh --seed

Scaffold a resource

Generate a model, migration, and resource controller in one command.

php artisan make:model Post -mcr

Clear & cache config

Clear stale caches in dev; cache for speed in production.

php artisan optimize:clear
php artisan config:cache

Queue worker

Processes queued jobs. In production, keep it alive with Supervisor.

php artisan queue:work --tries=3 --timeout=90

Add a package

Pull in a new Composer dependency.

composer require guzzlehttp/guzzle

Storage symlink

Expose storage/app/public under public/storage.

php artisan storage:link

Inspect routes & tinker

List every route, or open an interactive REPL.

php artisan route:list
php artisan tinker

Log in

Connect to the MySQL shell. You'll be prompted for the password.

mysql -u root -p

Create the database

Use utf8mb4 so emoji and all Unicode store correctly.

CREATE DATABASE azzok
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

Create a user & grant access

Give the app its own credentials rather than reusing root.

CREATE USER 'azzok'@'localhost' IDENTIFIED BY 'strong_password';
GRANT ALL PRIVILEGES ON azzok.* TO 'azzok'@'localhost';
FLUSH PRIVILEGES;

Back up (export)

Dump the whole database to a single SQL file.

mysqldump -u root -p azzok > azzok_backup.sql

Restore (import)

Load a dump back in. Create the empty database first.

mysql -u root -p azzok < azzok_backup.sql

Inspect tables

List tables in the current database and view a table's columns.

USE azzok;
SHOW TABLES;
DESCRIBE users;

Export a single table

Dump just one table instead of the whole database.

mysqldump -u root -p azzok users > users.sql

Reset a user password

Change a MySQL account password, then reload privileges.

ALTER USER 'azzok'@'localhost'
  IDENTIFIED BY 'new_password';
FLUSH PRIVILEGES;

Active connections

See what queries are currently running.

SHOW PROCESSLIST;

Check your setup

Verifies the SDK, toolchains, and connected devices.

flutter doctor -v

Install packages

Fetches dependencies listed in pubspec.yaml.

flutter pub get

Run the app

Launch on the default device, or target the browser with -d chrome.

flutter run
flutter run -d chrome

Reload while running

In the running session, press these keys in the terminal.

# r  = hot reload  (keep app state)
# R  = hot restart (reset state)
# q  = quit

Build a release APK

Produces an installable APK for direct distribution/testing.

flutter build apk --release

Build an App Bundle

The .aab format required for Google Play uploads.

flutter build appbundle --release

Clean a broken build

Clears build artifacts, then re-fetches packages — the fix for most stale-cache errors.

flutter clean
flutter pub get

Create a project

Scaffold a new Flutter app.

flutter create my_app

Upgrade & list devices

Update the SDK, then see connected targets.

flutter upgrade
flutter devices

Build for iOS

Compile a release build (run on macOS with Xcode).

flutter build ios --release

Nginx: test & reload

Validate the config before reloading — never reload a broken config.

sudo nginx -t && sudo systemctl reload nginx

Restart PHP-FPM

Apply PHP or opcache changes (adjust the version to match yours).

sudo systemctl restart php8.2-fpm

Supervisor: reload workers

Pick up new queue-worker config and restart all programs.

sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl restart all

Redis: health check

Confirm Redis responds, or watch live commands.

redis-cli ping
redis-cli monitor

Fix Laravel permissions

Give the web user ownership of the writable directories.

sudo chown -R www-data:www-data storage bootstrap/cache
sudo chmod -R 775 storage bootstrap/cache

Tail the app log

Follow the Laravel log live while reproducing an issue.

tail -f storage/logs/laravel.log
